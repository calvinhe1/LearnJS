LearnJS


Getting Started

Download the LearnJS.js and LearnJS.css files into your working directory, then link the LearnJS.js and LearnJS.css files to your webpage.

Example of integration of library into a basic webpage 
            
                <!DOCTYPE html>
                <html lang="en">
                    <head>
                        <meta charset="utf-8">
                        <title>Your title</title>  
                         <link rel="stylesheet" href="LearnJS.css"> 
                        <link rel="stylesheet" href="yourcssfile.css"> 
                        <script defer type="text/javascript" src="LearnJS.js"></script>
                        <script defer type="text/javascript" src="yourjsfile.js"></script> 
                        </head>
                        <body>
                        </body>
                        </html>

To get started with using the library, create a LearnJS object in your JS file.

Then, add a list container to hold the learning objectives that will be added to it.


Recommended: Include a hide list button, move list button, and search bar as well to improve the usability of the library (see example pages to see usefulness). 

            
                //Get LearnJS Object
                const learn =  LearnJS()

                //Add a list container
                learn.addList()

                //Add search bar for future searching of learning objectives by category
                learn.addSearchBar()

                //Add a hide button to hide/unhide the list of learning objectives
                learn.addHideButton()

                //Add a move button to move the list of learning objectives, so users can view it anywhere
                learn.addMoveButton()

API Object

LearnJS()

This object contains all of the necessary functions for the LearnJS library.


API Methods of LearnJS()

addList()

Create a list container to hold learning objectives.


Behaviours

  • Adds an event listener for users to add learning objectives when they click the plus button on the list container. Must submit the popup form to successfully add.

addObjective(difficultyLevel)

Add a learning objective with a difficulty level of difficultyLevel to the list container. If not specified, the default difficulty level is easy.


Arguments

  • difficultyLevel: a string that is either "easy", "intermediate", or "hard" to represent the difficulty level of the learning objective.

Behaviours

  • Adds an event listener for users to delete the learning objective when they click the delete button below the learning objective.
  • Adds an event listener for users to display a popup form when they right-click the learning objective to edit the details (difficulty level, category, title, description) of the learning objective.
  • Adds an event listener for users to view the description of a learning objective when they mouse over it (for non-empty description).
  • Adds an event listener to hide the description when users mouse out of the learning objective.
  • Adds an event listener for users to change the color of a learning objective to signify their current level of understanding to help keep track of their learning progress by clicking on the learning objective.
  • A popup form is displayed immediately when a user adds a learning objective through the event listener. If this form is closed without submission, the learning objective is deleted.

Return

  • Return the ID of the learning objective (ID starts at 1 and is incremented by 1 for each successive learning objective).

deleteObjective(objectiveNumber)

Delete the learning objective with ID objectiveNumber from the list container. (The first learning objective shown on the list has ID 1, the second has ID 2, etc.)


Arguments

  • objectiveNumber: an integer that is the ID of the objective to be deleted.

Behaviours

  • When an objective is deleted, all of the objective IDs that come after it shifts down by 1 to keep a consecutive list of objective IDs.

editTitle(objectiveNumber, title)

Edit the title of an objective with ID objectiveNumber in the list container.


Arguments

  • objectiveNumber: an integer that is the ID of the objective to be edited.
  • title: a string and the name of a learning objective that is displayed on top of the learning objective figure.

editDescription(objectiveNumber, description)

Edit the description of an objective number objectiveNumber in the list container.


Arguments

  • objectiveNumber: an integer that is the ID of the objective to be edited.
  • description: a string and the information of the learning objective that is displayed when a user mouses over the learning objective.

editDifficulty(objectiveNumber, difficultyLevel)

Edit the difficulty level of an objective objectiveNumber to difficultyLevel in the list container.


Arguments

  • objectiveNumber: an integer that is the ID of the objective to be edited.
  • difficultyLevel: a string that is either "easy", "intermediate", or "hard" to represent the difficulty level of the learning objective.

editCategory(objectiveNumber, category)

Edit the category of an objective number objectiveNumber in the list container.


Arguments

  • objectiveNumber: an integer that is the ID of the objective to be edited.
  • category: a string and the category of a learning objective, to be used for searching and filtering learning objectives by category.

changeColor(objectiveNumber, color)

Change the color of an objective with the ID objectiveNumber in the list container.


Arguments

  • objectiveNumber: an integer that is the ID of the objective to be changed in color.
  • color: a string that is one of red, yellow, or green; the colors to represent the understanding of an objective (red = no understanding, yellow = some understanding, green = complete understanding)

form(objectiveNumber, on)

Open/close the form for an objective with ID objectiveNumber in the list container.


Arguments

  • objectiveNumber: an integer that is the ID of the objective to open the form for.
  • on: either 0 or 1, if 1 the function opens the form, and if 0 it closes the form.

Behaviours

  • Must turn off the form, e.g. form(ID, 0) before opening another form. Do not call any other API DOM functions while the form is open (only call get methods).
  • Note: this method is typically used in the very end of the program, to simply open the form for the user when they initially visit the page (see editing examples page).

Form Behaviour

  • Adds an event listener for when the user types any key in the category field, showing a dropdown of categories that partially or fully matches with what was typed in. When the enter key is pressed, the recommended choices are hidden.
  • Adds an event listener for when the user clicks on a recommended category in the dropdown menu in the category field, selecting the category.
  • Adds an event listener for users to change the difficulty level, by selecting a different option in the difficulty field.
  • Adds an event listener for users to submit the form of a learning objective with any changes by clicking the submit button.
  • Adds an event listener for users to close the form to remove any changes that were added in the current form by clicking the close button.
  • While the form is open, users can only edit the form and submit or close it. They cannot for example add or delete learning objectives at this time.

addSearchBar()

Add a search bar to allow users to search and filter learning objectives by category.


Behaviours

  • Adds an event listener for when the user types any key in the category field, showing a dropdown of categories that partially or fully matches with what was typed in. When the enter key is pressed, the search is registered and the learning objectives matching the search show up. Note: entering nothing will show all the learning objectives.
  • Adds an event listener for when the user clicks a recommended category in the dropdown menu, acting as an enter key with the clicked category.

search(category)

Search for a category with the argument category, however, don't register the search, simply show the dropdown menu of recommended categories that partially or fully match category.


Arguments

  • category: a string and the category of learning objectives to search for.

Behaviours

  • Different capitalizations of the same category are shown in the dropdown menu, however, are considered the same category.

filter(category)

Register in the search of category, remove the dropdown menu of recommended categories and show only the learning objectives that matched category in the list container.


Arguments

  • category: a string and the category of learning objectives to filter on.

Behaviours

  • During a filtered view, only the learning objectives that are shown in the DOM can be manipulated through API methods that affect the DOM (e.g. deleteObjective, changeColor, editTitle, form, editCategory).
  • Calling addObjective() in a filtered view will by default set the category for that new objective to the currently searched category.
  • Reminder: the IDs of the learning objectives do not change during a filter, e.g. the first objective shown in the filtered view does not necessarily have an ID of 1. Recommended: make use of the getCategories() API method to determine the objective IDs (look at positions of returned array) that are currently shown in the DOM.
  • If the searched category is an empty string e.g. filter(""), then all the learning objectives are shown (default view is shown).
  • If the category for a learning objective changes to something different than category during a filtered view, it is removed from the DOM in the filtered view.

addHideButton()

Add a button that allows for the hiding/unhiding of the list container entirely. In reality, it is removed from the DOM and re-added when unhiding.


Behaviours

  • Add an event listener for when the user clicks on the hide/unhide list button.
  • Removes the popup form if it is open.

addMoveButton()

Add a button that allows for the movement of the list container entirely. When users are scrolled down and away from the list container, the list container can be moved to wherever the user is.


Behaviours

  • Add an event listener for when the user clicks on the move list button.
  • Removes the popup form if it is open.

getCategories()

Get all the categories for the current set of objectives.


Return

  • An array of all the categories of the learning objectives.

getTitles()

Get all the titles for the current set of objectives.


Return

  • An array of all the titles of the learning objectives.

getDescriptions()

Get all the descriptions for the current set of objectives.


Return

  • An array of all the descriptions of the learning objectives.

getDifficulties()

Get all the difficulties for the current set of objectives.


Return

  • An array of all the difficulty levels of the learning objectives.
